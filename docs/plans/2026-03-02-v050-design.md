# v0.5.0 Release — Design

**Date:** 2026-03-02
**Branch:** develop
**Status:** Approved
**Breaking:** Yes — fixed dirs default off, plist label renamed

## Overview

v0.5.0 is a breaking release that transforms asimov from a personal fork into a
publishable, independently maintained project. It ships 15 improvements across
distribution, performance, UX, config file support, and project hygiene.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| GitHub identity | `django23` | Homebrew tap, plist label, URLs |
| Fixed dirs default | OFF (breaking) | Build for the future; conservative sentinel-based ethos |
| Migration | UPGRADING.md + changelog | Explicit breaking version with upgrade guide |
| README tone | Minimal-technical, slightly warm | Between ripgrep and Homebrew |
| Curl install path | `~/.local/bin` (no sudo) | User-local, no privilege escalation |
| `du` parallelization | Skip | Double-du already fixed; diminishing returns |
| Config file | Include in v0.5.0 | Signature differentiating feature |

## 1. Distribution & Publishing (B, C, E, J)

### Homebrew Tap (B)

Create `django23/homebrew-tap` repo with a formula for asimov. Users install via:

    brew install django23/tap/asimov

Formula downloads a tagged release tarball, copies the `asimov` script to bin,
installs the plist. GitHub Actions updates the formula SHA on each release.

### Curl Installer (C)

New `scripts/install-remote.sh` that downloads the script + plist to
`~/.local/bin/asimov` and `~/Library/LaunchAgents/com.django23.asimov.plist`.
No sudo required. Plist `Program` path rewritten to `~/.local/bin/asimov`.

    curl -fsSL https://raw.githubusercontent.com/django23/asimov/main/scripts/install-remote.sh | bash

### Plist Label Rename (E)

Rename `com.stevegrunwell.asimov` → `com.django23.asimov` across:

| File | Change |
|---|---|
| `com.stevegrunwell.asimov.plist` | Rename file to `com.django23.asimov.plist`, update Label |
| `scripts/vars` | Update PLIST variable |
| `scripts/install.sh` | Update launchctl grep pattern |
| `scripts/uninstall.sh` | Update launchctl grep pattern |
| `tests/plist.bats` | Update filename, label assertions, comments |
| `README.md` | Update plist references |

Historical references in CHANGELOG.md (stevegrunwell/asimov#XX issue links) stay —
they're attribution to the original project's issues/PRs.

### Version Tag (J)

Bump `ASIMOV_VERSION` to `0.5.0`. Tag `v0.5.0` after all work is merged.

## 2. Performance (D)

### Remove tmutil isexcluded Redundancy (D)

The `mdfind` query in `build_find_skip_params()` already fetches all excluded paths
and prunes them from `find`. Paths reaching `exclude_paths_from_stdin()` should not
be excluded. Remove the `tmutil isexcluded` check from the default code path.

Under `--verbose`, keep a lightweight check so users can see what's already excluded.

This cuts ~30-50% of subprocess spawns per run.

Item I (parallelize du) is deferred — the double-du fix from the clean code sprint
already halved the cost, and diminishing returns don't justify the complexity.

## 4. New Flags & Tests (F, G, N)

### --quiet Flag (N)

Suppress all output except errors (stderr). No per-path messages, no summary.
Useful for launchd/cron.

Verbosity spectrum:

    --quiet     errors only (stderr)
    (default)   newly excluded paths + summary
    --verbose   all paths including already-excluded + summary

`--quiet` and `--verbose` are mutually exclusive. Both together → error, exit 1.

### format_size_kb Unit Tests (F)

New `tests/format.bats` testing boundary values:
- 0K, 1K, 1023K (KB range)
- 1024K = 1.0M (exact MB boundary)
- 1535K, 1536K (truncation behavior)
- 1048576K = 1.0G (exact GB boundary)
- Large values (2621440K = 2.5G)

### Flag Combination Tests (G)

- `--dry-run --verbose` — shows Would exclude + already excluded messages
- `--quiet --verbose` — exits 1 with error

## 5. Config File Support (H)

Incorporating the approved design from `docs/plans/2026-02-20-config-file-design.md`.

**Location:** `~/.config/asimov/config`
**Format:** INI-style with `[section]` headers, `key = value`, `#` comments.

```ini
[fixed_dirs]
enabled = true
extra = ~/.my-tool-cache

[sentinels]
extra = .build build.zig
disabled = node_modules package.json
```

**Defaults (no config file):** Fixed dirs OFF. All built-in sentinels active.

**New function:** `load_config()` — awk-based INI parser. Called once after
`ASIMOV_ROOT` is resolved. Populates:

| Variable | Type | Default |
|---|---|---|
| `ASIMOV_CONFIG_FIXED_DIRS_ENABLED` | string | `false` |
| `ASIMOV_CONFIG_EXTRA_FIXED_DIRS` | array | `()` |
| `ASIMOV_CONFIG_EXTRA_SENTINELS` | array | `()` |
| `ASIMOV_CONFIG_DISABLED_SENTINELS` | array | `()` |

**Modified behavior:**
- `build_find_vendor_params()` skips disabled sentinels, appends extras
- Fixed dirs loop guarded by `ASIMOV_CONFIG_FIXED_DIRS_ENABLED == true`
- Extra fixed dirs always processed regardless of enabled flag

**Error handling:** Silent on missing file, unknown sections/keys, malformed lines.

**Tests:** 8 new tests using `write_config()` helper in test_helper.bash.
